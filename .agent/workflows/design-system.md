# Design System

I will help you create or analyze a design system for your project.

## Guardrails
- Analyze existing styles before creating new
- Keep tokens consistent and documented
- Use semantic naming
- Consider accessibility

## Steps

### 1. Understand Scope
Ask clarifying questions:
- Creating new or analyzing existing?
- What components are needed?
- Brand guidelines to follow?
- Accessibility requirements?

### 2. Analyze Existing
If updating:
- Review current colors, typography, spacing
- Identify inconsistencies
- Note pain points

### 3. Define Tokens
Create core tokens:

**Colors:**
- Primary, secondary, accent
- Semantic (success, warning, error)
- Neutrals (gray scale)

**Typography:**
- Font families
- Size scale
- Weight scale
- Line heights

**Spacing:**
- Base unit (4px or 8px)
- Scale (xs, sm, md, lg, xl)

**Other:**
- Border radius
- Shadows
- Breakpoints

### 4. Create Components
Build primitives:
- Button (variants, sizes)
- Input (states)
- Card
- Typography components

### 5. Document
Create documentation:
- Token reference
- Component examples
- Usage guidelines
- Do's and don'ts

## Principles
- Consistency over perfection
- Semantic naming
- Mobile-first
- Accessibility built-in
